CLR Chapter 5-1 - Primitive, Reference, Value Types

**日期**: 2026-03-30
2024-01-01 · 3 min · shiwenbot |  建议修改

日期: 2026-03-30 章节: CLR via C# Chapter 5-1 时长: 约50分钟

核心概念速览#

  • 基元类型: int = Int32，固定32位，不随平台变化
  • Checked/Unchecked: 默认不检查溢出（性能优先），checked关键字可显式开启
  • Decimal特殊性: 非CLR基元类型，无IL指令，总是检查溢出
  • 引用类型: 在堆上分配，变量存储地址
  • 值类型: 通常在栈上分配，变量直接存储数据
  • 赋值语义: 引用类型复制引用，值类型字段逐字段复制
  • 值类型继承限制: 所有值类型都是sealed，不能被继承

恍然大悟时刻（重点！）#

1. int 永远是32位#

我的疑问: int 在32位和64位系统上分别是多少位？

探索过程:

  • 我一开始猜是4字节和8字节
  • 三月七纠正说 int 永远是32位
  • 原来 int 只是 Int32 的语法糖，C#编译器自动添加 using int = System.Int32;

最终解释:

  • int = System.Int32，固定32位（4字节）
  • 与平台无关！不管是在32位还是64位系统上，都是32位
  • 这样设计是为了跨平台一致性和跨语言互操作
  • 作者Jeff Richter建议使用 Int32 而不是 int，避免平台误解

类比/记忆点:

  • 就像"米"的定义——不管你在哪个国家，1米就是1米，不会变

2. Checked vs Unchecked#

我的疑问: 怎么让C#在溢出时抛出异常？

探索过程:

  • 三月七告诉我用 checked 关键字
  • 但默认是 unchecked（不检查）

最终解释:

  • 默认 unchecked: 为了性能，溢出静默发生（如 Byte 100 + 200 = 44）
  • IL指令差异:
    • add: 直接相加，不检查（快）
    • add.ovf: 相加并检查溢出，溢出抛异常（慢）
  • 使用方式:
    Byte b = 100;
b = checked((Byte)(b + 200));  // 抛出 OverflowException

checked {  // 检查代码块内所有运算
    Byte b2 = 100;
    b2 = (Byte)(b2 + 200);  // 也抛异常
}
  • Decimal 特殊: 没有IL指令，checked/unchecked 对它无效，总是检查溢出

类比/记忆点:

  • 像开车时的"限速提醒"——默认不提醒（省事儿），但可以开启提醒（安全）

3. 引用类型 vs 值类型的赋值语义#

我的疑问: r2 = r1 和 v2 = v1 有什么区别？

探索过程:

  • 三月七让我画内存图
  • 引用类型变量存的是地址，值类型变量存的是实际数据

最终解释:

class SomeRef { public int x; }  // 引用类型
struct SomeVal { public int x; }  // 值类型

SomeRef r1 = new SomeRef(); r1.x = 5;
SomeVal v1 = new SomeVal(); v1.x = 5;

SomeRef r2 = r1;  // r2 和 r1 指向同一个堆对象！
SomeVal v2 = v1;  // v2 是 v1 的完整独立副本

r1.x = 8;  // r2.x 也变成 8（因为指向同一对象）
v1.x = 9;  // v2.x 还是 5（完全独立）

内存图景:

类比/记忆点:

  • 引用类型赋值 = “共享钥匙”（两个人用同一把钥匙开同一扇门）
  • 值类型赋值 = “复制整栋房子”（两个人各有自己的房子）

4. 值类型为什么不能继承#

我的疑问: 为什么值类型不能被继承？

探索过程:

  • 三月七反问我：如果值类型可以继承，数组会出什么问题？
  • 我想到了：数组元素大小不确定了！

最终解释:

// 假设值类型可以继承
struct Base { public int x; }  // 4字节
struct Derived : Base { public int y; }  // 8字节

Base[] array = new Base[10];  // 每个元素该多大？
array[0] = new Derived();     // 8字节的数据塞进4字节的空间？
  • 数组需要固定元素大小: 为了随机访问（O(1)），数组必须知道每个元素多大
  • 如果值类型可继承: 大小不确定，数组无法分配内存
  • 引用类型可以继承: 因为数组存的是地址（固定8字节），不是对象本身

类比/记忆点:

  • 像火车车厢——每节车厢必须一样长，不然火车跑不起来

5. ValueType 的"魔法"#

我的疑问: ValueType 是怎么让子类变成值类型的？

探索过程:

  • 三月七问：ValueType 自己是什么类型？
  • 我答：引用类型！
  • 那为什么它的子类是值类型？

最终解释:

  • CLR 特殊处理: 不是 ValueType 类本身实现的，而是 CLR 的类型系统规则
  • 规则: 如果类型的继承链上有 ValueType（除了ValueType自己），就是值类型
  • 分配位置由CLR决定: 值类型在栈上（通常），引用类型在堆上
  • ValueType 的作用: 提供默认实现（如重写了Equals做字段比较）

类比/记忆点:

  • 像"VIP会员卡"——有了这张卡，CLR给你特殊待遇（栈分配）
