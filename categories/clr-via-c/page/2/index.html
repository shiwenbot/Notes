CLR Chapter 9 - Parameters

日期: 2026-04-03 章节: CLR via C# Chapter 9 - Parameters 时长: 约50分钟（分两次课：9-1 + 9-2） 核心概念速览 默认参数版本陷阱: 编译器将默认值嵌入调用方 IL，只改定义不重编调用方会拿到旧值 命名参数求值顺序: 语法可乱序，但编译器按物理位置从左到右求值 var 本质: 纯编译期类型推断语法糖，IL 中无 var；dynamic 是 Object + DynamicAttribute + 运行时绑定 ref/out 底层: IL 层几乎相同（仅 1 bit 差异），本质都是传地址 ref/out C# 语义: ref 要求调用方初始化；out 要求被调用方赋值，赋值前不可读取 ref 引用类型类型安全: 不允许 ref String 传 ref Object，防止方法体替换引用为不兼容类型 params 底层: ParamArrayAttribute，编译器在调用点自动构造数组 params 性能陷阱: 每次调用都有堆分配；高频场景用 1/2/3/4 参数显式重载优化（如 String.Concat） 参数/返回设计原则: 参数用最弱类型（IEnumerable<T>），返回用最强类型（FileStream） 恍然大悟时刻（重点！） 1. 默认参数的 “版本陷阱” 我的疑问: 默认参数改了默认值，不重新编译调用方会怎样？ 探索过程: 三月七拿 const 的跨程序集陷阱做类比。 varsity 都是"编译时 embedding"——const 是嵌入值，默认参数是嵌入默认值。 ...
